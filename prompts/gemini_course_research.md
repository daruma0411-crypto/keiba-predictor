# Gemini Deep Research用プロンプト — QMCコースプロファイル生成

以下の4コースそれぞれをGeminiディープリサーチに投げてください。
返ってきたJSONをqmc_courses.pyのCOURSE_PROFILESに追加します。

---

## プロンプト（共通テンプレート）

以下のプロンプトの `{コース名}` `{直線距離}` `{コース特徴}` を各コースに差し替えて使ってください。

---

### コース1: 中山芝2000m（皐月賞コース）

```
あなたは競馬のコース分析の専門家です。
中山競馬場 芝2000m（内回り）について、直近3年間（2023-2025年）のレースデータを分析し、以下の統計を出してください。

【分析項目】
1. 脚質別成績（逃げ/先行/差し/追込それぞれの勝率・連対率・複勝率）
2. 枠順別成績（1-4枠 vs 5-8枠の勝率・複勝率）
3. 内枠×先行脚質の成績
4. 外枠×差し/追込脚質の成績
5. 内枠×差し脚質の包まれ率（掲示板外の比率）
6. ペース傾向（ハイペース/ミドル/スローの比率と、各ペース時の脚質別成績）
7. 出遅れ・不利の発生頻度（概算）
8. コース形態の特徴（直線距離310m、急坂、小回り等がレース結果にどう影響するか）
9. 重賞（特に皐月賞）でのバイアス傾向

【出力フォーマット】
分析結果を元に、以下のJSON形式のQMCシミュレーション係数を提案してください。
各係数の意味は以下の通りです：

{
  "name": "中山芝2000m",
  "straight": 310,  // 直線距離(m)
  "pace_base_per_runner": 0.XX,  // 逃げ馬1頭増あたりのペース上昇係数 (0.15-0.40)
  "pace_noise": 0.XX,  // ペースのランダム変動幅 (0.10-0.25)
  "style_bonus": {
    "nige": +0.XX,    // 逃げのボーナス/ペナルティ (-0.10 ~ +0.15)
    "senkou": +0.XX,  // 先行
    "sashi": -0.XX,   // 差し
    "oikomi": -0.XX   // 追込
  },
  "gate_bias": {
    "inner_senkou": -0.XX,  // 内枠先行の有利度 (負=有利)
    "outer_sashi": +0.XX,   // 外枠追込の不利度 (正=不利)
    "inner_block": +0.XX    // 内枠差しの包まれリスク (正=不利)
  },
  "trouble_rate": 0.XX,     // 出遅れ・不利の発生確率 (0.03-0.08)
  "trouble_penalty": 0.XX,  // 不利発生時のペナルティ (0.10-0.20)
  "noise_scale": 0.XX       // 個体ノイズの大きさ (0.01-0.04)
}

係数の決め方の指針：
- style_bonus: 脚質別の複勝率の偏りから算出。全脚質の複勝率が均等なら全て0に近い。逃げの複勝率が40%なら+0.10程度。
- gate_bias: 枠順×脚質のクロス集計から算出。内枠先行の複勝率が外枠先行より10%以上高ければ inner_senkou=-0.02程度。
- pace_noise: スロー比率が高いコースは低め(0.10-0.13)、ハイペースが多いコースは高め(0.18-0.25)。
- trouble_rate: 多頭数・小回りコースは高め(0.06-0.08)、大箱コースは低め(0.03-0.05)。

必ず統計的根拠を示し、なぜその係数にしたか理由を付けてください。
```
